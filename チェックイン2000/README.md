Check-in Generator（チェックイン質問特化型アプリ）会議、ワークショップ、1on1、チームビルディングなどの冒頭で行う「チェックイン」を活性化させるための、質問生成・提示ツールです。2000種類の洗練された質問データベースから、その日の目的や参加者の関係性に合った「最高の1問」を直感的に選び出すことができます。🎯 プロジェクトの目的ファシリテーターやリーダーが、毎回のミーティングで質の高い「問い」を簡単に用意できるようにする。参加者の心理的安全性を高め、相互理解を深めるためのきっかけを提供する。サーバーやバックエンドを用意せず、フロントエンドのみで完結する軽量で高速なシステムを実現する。✨ 主な機能ランダム質問表示: 2000問のデータベースから、被りなくランダムに質問を表示します。2段階フィルタリング: 「ジャンル（大分類）」と「サブジャンル（小分類）」の組み合わせで、目的に合った質問に絞り込むことができます。クリップボードコピー: 表示された質問をワンクリックでコピーし、チャットツール等に簡単に貼り付けられます。フラッシュカードUI: 画面共有時にも見やすい、シンプルで視認性の高いカード型デザインを採用しています。📁 質問の分類体系（ジャンル / サブジャンル）以下の6つの大ジャンルと、それぞれに紐づくサブジャンルを用意しています。日常・振り返り (嬉しかったこと / 失敗・反省 / 学び・気づき / 感謝)食・ライフスタイル (外食・グルメ / 自炊・おうちご飯 / 習慣・ルーティン / 買い物・アイテム)趣味・エンタメ (映画・ドラマ / 音楽 / 旅行・お出かけ)価値観・働き方 (仕事のこだわり / 人生観・座右の銘 / 人間関係・コミュニケーション)もしもの話 (超能力・魔法 / 究極のシチュエーション)究極の2択 (ライフスタイル / 食の好み / 価値観)🛠️ 技術スタックFrontend: HTML5, CSS3, Vanilla JavaScriptDatabase: JSON (JavaScriptオブジェクト配列として保持)Hosting: GitHub Pages (推奨)🚀 セットアップと公開手順本システムはバックエンドを必要としないため、以下の手順ですぐに公開・運用が可能です。1. リポジトリの準備GitHubで新しいリポジトリを作成します。以下の4つのファイルを作成し、リポジトリに追加（コミット＆プッシュ）します。index.html (UI)style.css (スタイル・アニメーション)app.js (メインロジック)json.js (2000問の質問データ)2. データ（json.js）のセットアップ付属の checkin_questions_natural_2000.csv データを、JavaScriptのオブジェクト配列形式に変換し、json.js に記述します。// json.js の記述例
const questionsData = [
  { id: 1, question: "最近1ヶ月で、一番嬉しかったことを教えてください。", genre: "日常・振り返り", sub_genre: "嬉しかったこと" },
  // ...残り1999件
];
3. GitHub Pagesでの公開リポジトリの [Settings] タブを開きます。左側メニューの [Pages] をクリックします。「Build and deployment」の「Source」を Deploy from a branch に設定します。「Branch」で main（または master）を選択し、/(root) のまま [Save] をクリックします。数分後、画面上部に表示されるURL（https://[あなたのID].github.io/[リポジトリ名]/）にアクセスすれば公開完了です。📂 ファイル構成📦 checkin-generator
 ┣ 📜 index.html    # メイン画面（UI）
 ┣ 📜 style.css     # デザイン・アニメーション設定
 ┣ 📜 app.js        # 絞り込み、ランダム抽出、コピー機能のロジック
 ┗ 📜 json.js       # 2000問の質問データベース（配列）
🤝 カスタマイズについて特定のチーム内だけで使う「内輪ネタ」の質問や、プロジェクトに特化した質問を追加したい場合は、json.js の配列にデータを追記するだけで簡単にカスタマイズ可能です。
